# Trace Miner

Use this skill to turn public trace evidence into operational Codex behavior.

## Workflow

1. Confirm the source is public and record its URL, license if visible, and date checked.
2. Extract behavior patterns only: planning, evidence gathering, tool discipline, validation, review, handoff, and recovery.
3. Do not copy leaked prompts, hidden instructions, raw chain-of-thought, secrets, or proprietary code.
4. Put long notes in `docs/source-index.md` or `docs/extracted-patterns.md`.
5. Convert useful patterns into a skill, hook, ledger field, or eval criterion.
6. Validate the change with the before/after scorecard when behavior changes.
